
Support (use whenever needed): Slides ‚Ä¢ Answer Keys ‚Ä¢ Prompt Templates.

Rule: ONLY move down when you can explain the current step in your own words.

Key ideas:
- Move clockwise; don‚Äôt rush to puzzles.
- Reflection (teaching it back) locks in learning.
- Mastery check loops you back only where needed (targeted review, not full restart).
- Support artifacts (slides, keys, prompts) assist mentors without taking away student discovery.

Plain-text fallback: Concept ‚Üí Examples ‚Üí Homework ‚Üí Tailored Puzzles ‚Üí Reflection ‚Üí (Mastery?) ‚Üí Mini Project (else loop back).

## Tips for Success! üí°

1. **Be Curious!**
  - Change example code on purpose‚Äîpredict the output first.
  - Run it. Were you right? If not, why?
2. **Practice Regularly**
  - Small daily reps > long weekly cram.
  - Finish homework before peeking at a key.
3. **Use Puzzles Strategically**
  - They should feel like ‚Äúmore of the same,‚Äù not ‚Äúbrand new.‚Äù If not, step back.
4. **Explain Out Loud**
  - If you can narrate each line‚Äôs job, you own it.
5. **Debug Mindset**
  - Change ONE thing. Re-run. Observe. Repeat.

Mentor Cue: Ask ‚ÄúWhat did you expect?‚Äù before giving hints.
